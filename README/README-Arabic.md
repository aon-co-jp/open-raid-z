# open-raid-z

مشروع نظام ملفات تجريبي لنظام Windows يحافظ على توافق شبه كامل مع NTFS/exFAT، مع توفير ميزات بأسلوب ZFS (مجاميع اختبارية ذاتية الإصلاح، مجمّعات تخزين، النسخ عند الكتابة، اللقطات/الاستنساخ) إلى جانب RAID0/1/5/6/10/Z2/Z3.

## المكوّنات

| المكوّن | الدور |
|---|---|
| `openzfs-winfsp-bridge` | أجهزة vdev لـ RAID-Z/RAID0-10، مجمّع التخزين، طبقة توافق قوائم التحكم بالوصول NTFS/سمات exFAT، تركيب حقيقي عبر WinFsp |
| `zfs-accel-hlsl` | تفريغ حساب التكافؤ إلى مسرّعات عتادية GPU/NPU (DirectX 12 Compute + DirectML) |
| `openruno-installer` | مثبّت مبني على Tauri: اكتشاف العتاد، معالج تهيئة zpool، مستشار تهيئة على طراز Copilot |

## الميزات الرئيسية

- **دعم كامل لسلسلة RAID**: RAID0 / RAID1 (مرآة) / RAID5 / RAID6 / RAID10 (مرايا مقسّمة) / RAID-Z2 / RAID-Z3
- **تقسيم القرص وإعادة استخدامه**: تقسيم قرص فعلي واحد واستخدام نصفه كعضو في مرآة بينما ينضم النصف الآخر إلى مصفوفة RAID6/Z2 منفصلة
- **مجاميع اختبارية ذاتية الإصلاح، النسخ عند الكتابة، اللقطات/الاستنساخ**: تحاكي أسلوب ZFS
- **التوافق مع NTFS**: ترجمة قوائم التحكم بالوصول (NFSv4 ⇔ NTFS)، وربط UID/GID ⇔ SID (ربط حتمي قائم على RID لنطاقات SAM المحلية/AD)
- **التوافق مع exFAT**: تحويل سمات الملفات والطوابع الزمنية، ودعم الملفات/الوحدات التخزينية التي تتجاوز 4 جيجابايت
- **تسريع عتادي عبر GPU/NPU**: يتم فعليًا إرسال حساب تكافؤ RAID-Z1/Z2 عبر DirectX 12 Compute + DirectML (يعود تلقائيًا إلى المعالج عند عدم وجود عتاد)
- **مستشار تهيئة على طراز Copilot**: يوصي بمستوى RAID بناءً على تخطيط الأقراص والمسرّع وعدد أنوية المعالج (نسخة أولى قائمة على قواعد استدلالية؛ كما يتوفر هيكل أساسي لاكتشاف نموذج لغوي محلي)
- **تركيب حقيقي عبر WinFsp (نموذج أولي)**: يمكن تركيبه فعليًا كحرف محرك أقراص في Windows (حاليًا نسخة بسيطة بملف واحد فقط)
- **دعم متعدد اللغات**: يستخدم المثبّت اليابانية افتراضيًا مع مبدّل لغة في الواجهة، قابل للتغيير أيضًا بعد التثبيت

## القيود الحالية (مرحلة النموذج الأولي)

- يدعم تركيب WinFsp فقط مساحة أسماء مسطحة (ملف واحد ثابت `\pool.dat` في الجذر). لا يوجد بعد تسلسل هرمي للمجلدات أو ملفات متعددة.
- يجب أن تكون عمليات القراءة/الكتابة متوافقة مع حدود كتلة (chunk) مجموعة البيانات.
- يدعم `Pool` كلاً من `RaidZVdev` و`Raid10Vdev`، لكن تكامل RAID10 مع واجهة برمجة مجموعة البيانات لا يزال سطحيًا في بعض الأجزاء.

## البناء والاختبار

```powershell
cd open-runo-zfs-source/openzfs-winfsp-bridge
cargo test --no-default-features        # بدون تركيب WinFsp
cargo test --features winfsp-backend    # مع التركيب الحقيقي عبر WinFsp (يتطلب بيئة تشغيل WinFsp)
```

يجب تثبيت بيئة تشغيل WinFsp (https://winfsp.dev/) على النظام (يتم تضمين ملفات ترويسة SDK المستخدمة أثناء البناء تلقائيًا، لذا لا حاجة لتثبيت منفصل لمكوّن المطوّرين).
