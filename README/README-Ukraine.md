# open-raid-z

Експериментальний проєкт файлової системи для Windows, що зберігає майже повну сумісність із NTFS/exFAT, надаючи функції у стилі ZFS (самовідновлювані контрольні суми, пули зберігання, копіювання при записі, знімки/клони) поряд із RAID0/1/5/6/10/Z2/Z3.

## Компоненти

| Компонент | Роль |
|---|---|
| `openzfs-winfsp-bridge` | vdev'и RAID-Z/RAID0-10, пул зберігання, рівень сумісності ACL NTFS/атрибутів exFAT, справжнє монтування через WinFsp |
| `zfs-accel-hlsl` | Вивантаження обчислення парності на апаратні прискорювачі GPU/NPU (DirectX 12 Compute + DirectML) |
| `openruno-installer` | Інсталятор на Tauri: виявлення обладнання, майстер ініціалізації zpool, консультант з конфігурації у стилі Copilot |

## Основні можливості

- **Повна серія RAID**: RAID0 / RAID1 (дзеркало) / RAID5 / RAID6 / RAID10 (дзеркала зі стрипінгом) / RAID-Z2 / RAID-Z3
- **Поділ диска на розділи та повторне використання**: розділити один фізичний диск і використати одну половину як учасника дзеркала, а іншу — як учасника окремого масиву RAID6/Z2
- **Самовідновлювані контрольні суми, копіювання при записі, знімки/клони**: імітують підхід ZFS
- **Сумісність із NTFS**: перетворення ACL (NFSv4 ⇔ NTFS), відповідність UID/GID ⇔ SID (детермінована відповідність на основі RID для локальних доменів SAM/AD)
- **Сумісність з exFAT**: перетворення атрибутів файлів і часових міток, підтримка файлів/томів понад 4 ГБ
- **Апаратне прискорення GPU/NPU**: обчислення парності RAID-Z1/Z2 реально виконується через DirectX 12 Compute + DirectML (автоматичний відкат на CPU за відсутності обладнання)
- **Консультант з конфігурації у стилі Copilot**: рекомендує рівень RAID на основі конфігурації дисків, прискорювача та кількості ядер CPU (перша евристична версія; також реалізовано каркас для виявлення локальної LLM)
- **Справжнє монтування через WinFsp (прототип)**: може бути реально змонтований як буква диска Windows (наразі мінімальна збірка з одним файлом)
- **Багатомовна підтримка**: інсталятор за замовчуванням використовує японську мову з перемикачем мови інтерфейсу, який можна змінити й після встановлення

## Поточні обмеження (стадія прототипу)

- Монтування через WinFsp підтримує лише плаский простір імен (один фіксований файл `\pool.dat` у корені). Ієрархії каталогів і кількох файлів поки немає.
- Читання/запис мають бути вирівняні за межею блоку (chunk) набору даних.
- `Pool` підтримує як `RaidZVdev`, так і `Raid10Vdev`, але інтеграція RAID10 з API набору даних місцями ще поверхнева.

## Збірка та тестування

```powershell
cd open-runo-zfs-source/openzfs-winfsp-bridge
cargo test --no-default-features        # без монтування WinFsp
cargo test --features winfsp-backend    # зі справжнім монтуванням WinFsp (потрібне середовище виконання WinFsp)
```

Середовище виконання WinFsp (https://winfsp.dev/) має бути встановлене в системі (заголовки SDK, що використовуються під час збірки, постачаються автоматично, тому окреме встановлення компонента для розробників не потрібне).
