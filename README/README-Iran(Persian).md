# open-raid-z

یک پروژه‌ی آزمایشی سیستم فایل برای ویندوز که سازگاری تقریباً کامل با NTFS/exFAT را حفظ می‌کند و ویژگی‌هایی به سبک ZFS (چک‌سام‌های خودترمیم‌شونده، استخرهای ذخیره‌سازی، کپی هنگام نوشتن، عکس‌های لحظه‌ای/کلون‌ها) را در کنار RAID0/1/5/6/10/Z2/Z3 ارائه می‌دهد.

## اجزا

| جزء | نقش |
|---|---|
| `openzfs-winfsp-bridge` | vdevهای RAID-Z/RAID0-10، استخر ذخیره‌سازی، لایه‌ی سازگاری ACL در NTFS/ویژگی‌های exFAT، سوارکردن واقعی از طریق WinFsp |
| `zfs-accel-hlsl` | واگذاری محاسبه‌ی توازن (parity) به شتاب‌دهنده‌های سخت‌افزاری GPU/NPU (DirectX 12 Compute + DirectML) |
| `openruno-installer` | نصب‌کننده‌ی مبتنی بر Tauri: شناسایی سخت‌افزار، جادوگر مقداردهی اولیه‌ی zpool، مشاور پیکربندی به سبک Copilot |

## ویژگی‌های کلیدی

- **پشتیبانی کامل از سری RAID**: RAID0 / RAID1 (آینه‌ای) / RAID5 / RAID6 / RAID10 (آینه‌های نواری) / RAID-Z2 / RAID-Z3
- **تقسیم‌بندی و استفاده‌ی مجدد از دیسک**: تقسیم یک دیسک فیزیکی و استفاده از نیمی از آن به‌عنوان عضو یک آرایه‌ی آینه‌ای، در حالی که نیمه‌ی دیگر به یک آرایه‌ی جداگانه‌ی RAID6/Z2 می‌پیوندد
- **چک‌سام‌های خودترمیم‌شونده، کپی هنگام نوشتن، عکس‌های لحظه‌ای/کلون‌ها**: شبیه‌سازی رویکرد ZFS
- **سازگاری با NTFS**: ترجمه‌ی ACL (NFSv4 ⇔ NTFS)، نگاشت UID/GID ⇔ SID (نگاشت قطعی مبتنی بر RID برای دامنه‌های محلی SAM/AD)
- **سازگاری با exFAT**: تبدیل ویژگی‌های فایل و برچسب‌های زمانی، پشتیبانی از فایل‌ها/حجم‌های بزرگ‌تر از ۴ گیگابایت
- **شتاب‌دهی سخت‌افزاری GPU/NPU**: محاسبه‌ی توازن RAID-Z1/Z2 واقعاً از طریق DirectX 12 Compute + DirectML اجرا می‌شود (در صورت نبود سخت‌افزار، به‌طور خودکار به CPU بازمی‌گردد)
- **مشاور پیکربندی به سبک Copilot**: بر اساس چیدمان دیسک‌ها، شتاب‌دهنده و تعداد هسته‌های CPU، سطح RAID مناسب را پیشنهاد می‌دهد (نسخه‌ی اولیه‌ی اکتشافی؛ یک ساختار پایه برای شناسایی مدل زبانی محلی نیز فراهم شده است)
- **سوارکردن واقعی از طریق WinFsp (نمونه‌ی اولیه)**: می‌تواند واقعاً به‌عنوان یک حرف درایو در ویندوز سوار شود (در حال حاضر یک نسخه‌ی ساده‌ی تک‌فایلی)
- **پشتیبانی چندزبانه**: نصب‌کننده به‌طور پیش‌فرض از ژاپنی استفاده می‌کند و دارای یک تعویض‌کننده‌ی زبان در رابط کاربری است که پس از نصب نیز قابل تغییر است

## محدودیت‌های فعلی (مرحله‌ی نمونه‌ی اولیه)

- سوارکردن WinFsp فقط از یک فضای نام مسطح پشتیبانی می‌کند (یک فایل ثابت `\pool.dat` در ریشه). هنوز سلسله‌مراتب پوشه یا چند فایل پشتیبانی نمی‌شود.
- خواندن/نوشتن باید با مرز بلوک (chunk) مجموعه‌داده هم‌تراز باشد.
- `Pool` هم از `RaidZVdev` و هم از `Raid10Vdev` پشتیبانی می‌کند، اما یکپارچگی RAID10 با API مجموعه‌داده در برخی جاها هنوز سطحی است.

## ساخت و آزمایش

```powershell
cd open-runo-zfs-source/openzfs-winfsp-bridge
cargo test --no-default-features        # بدون سوارکردن WinFsp
cargo test --features winfsp-backend    # با سوارکردن واقعی WinFsp (به زمان اجرای WinFsp نیاز دارد)
```

زمان اجرای WinFsp (https://winfsp.dev/) باید روی سیستم نصب باشد (هدرهای SDK استفاده‌شده در زمان ساخت به‌طور خودکار همراه پروژه ارائه می‌شوند، بنابراین نیازی به نصب جداگانه‌ی مؤلفه‌ی توسعه‌دهنده نیست).
