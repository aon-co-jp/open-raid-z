# open-raid-z

Экспериментальный проект файловой системы для Windows, сохраняющий почти полную совместимость с NTFS/exFAT и предоставляющий функции в стиле ZFS (самовосстанавливающиеся контрольные суммы, пулы хранения, копирование при записи, снимки/клоны) наряду с RAID0/1/5/6/10/Z2/Z3.

## Компоненты

| Компонент | Роль |
|---|---|
| `openzfs-winfsp-bridge` | vdev'ы RAID-Z/RAID0-10, пул хранения, слой совместимости ACL NTFS/атрибутов exFAT, настоящее монтирование через WinFsp |
| `zfs-accel-hlsl` | Разгрузка вычисления четности на аппаратные ускорители GPU/NPU (DirectX 12 Compute + DirectML) |
| `openruno-installer` | Установщик на Tauri: обнаружение оборудования, мастер инициализации zpool, консультант по конфигурации в стиле Copilot |

## Основные возможности

- **Полная серия RAID**: RAID0 / RAID1 (зеркало) / RAID5 / RAID6 / RAID10 (чередующиеся зеркала) / RAID-Z2 / RAID-Z3
- **Разбиение диска на разделы и повторное использование**: разделить один физический диск и использовать одну половину как участник зеркала, а другую — как участник отдельного массива RAID6/Z2
- **Самовосстанавливающиеся контрольные суммы, копирование при записи, снимки/клоны**: имитируют подход ZFS
- **Совместимость с NTFS**: преобразование ACL (NFSv4 ⇔ NTFS), сопоставление UID/GID ⇔ SID (детерминированное сопоставление на основе RID для локальных доменов SAM/AD)
- **Совместимость с exFAT**: преобразование атрибутов файлов и меток времени, поддержка файлов/томов объёмом более 4 ГБ
- **Аппаратное ускорение GPU/NPU**: вычисление четности RAID-Z1/Z2 реально выполняется через DirectX 12 Compute + DirectML (автоматический откат на CPU при отсутствии оборудования)
- **Консультант по конфигурации в стиле Copilot**: рекомендует уровень RAID на основе конфигурации дисков, ускорителя и количества ядер CPU (первая эвристическая версия; также реализован каркас для обнаружения локальной LLM)
- **Настоящее монтирование через WinFsp (прототип)**: может быть реально смонтирован как буква диска Windows (в настоящее время минимальная сборка с одним файлом)
- **Многоязычная поддержка**: установщик по умолчанию использует японский язык с переключателем языка интерфейса, который можно изменить и после установки

## Текущие ограничения (стадия прототипа)

- Монтирование через WinFsp поддерживает только плоское пространство имён (один фиксированный файл `\pool.dat` в корне). Иерархии каталогов и нескольких файлов пока нет.
- Чтение/запись должны быть выровнены по границе блока (chunk) набора данных.
- `Pool` поддерживает как `RaidZVdev`, так и `Raid10Vdev`, но интеграция RAID10 с API набора данных местами всё ещё поверхностна.

## Сборка и тестирование

```powershell
cd open-runo-zfs-source/openzfs-winfsp-bridge
cargo test --no-default-features        # без монтирования WinFsp
cargo test --features winfsp-backend    # с настоящим монтированием WinFsp (требуется среда выполнения WinFsp)
```

Среда выполнения WinFsp (https://winfsp.dev/) должна быть установлена в системе (заголовки SDK, используемые при сборке, поставляются автоматически, поэтому отдельная установка компонента для разработчиков не требуется).
